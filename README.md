## В ходе выполнения задания Фибоначи мы использовали следующие методы отладки:

- Пошаговое выполнение (Step Over и Step Into) для анализа работы цикла.
- Мониторинг значений переменных через окно "Локальные".
- Продолжение выполнения до следующего ключевого места (Continue).
- Анализ логики и поведения программы на основе полученных данных.

## В ходе выполнения задания Галактики мы использовали следующие средства и методы отладки в Microsoft Visual Studio:

- **Запуск программы в режиме отладки (F5)** — позволяющий запускать приложение с отладчиком и приостанавливать выполнение в любой момент.
- **Точки останова (Breakpoints)** — установка точек остановки в коде с помощью щелчка слева от строки или клавиши F9, чтобы остановить выполнение программы и проверить состояние переменных.
- **Пошаговое выполнение (Step Over, Step Into, Step Out)** — позволяющее подробно проследить выполнение кода по строкам или внутри функций, что помогает выявить точное место возникновения ошибок.
- **Просмотр значений переменных** — через всплывающие подсказки, окна Watch, Autos и Locals, что помогает отслеживать изменения данных во время выполнения.
- **Обработка исключений** — автоматическая перенаправка выполнения к коду обработки ошибок при возникновении исключений, что помогает быстро локализовать проблему.

Эти средства значительно упростили процесс поиска и устранения ошибок и повысили эффективность отладки.

## В ходе выполнения задания Буквы мы использовали следующие функции отладчика Visual Studio:

- **Запуск приложения с подключённым отладчиком (F5)** — запускал программу, чтобы наблюдать выполнение и получать вывод в консоль.
- **Установка точек останова** — поставил точки останова, чтобы приостановить выполнение программы в нужных местах, например, в строке `name += letters[i];` в цикле.
- **Пошаговое выполнение кода (F10 — шаг с обходом, F11 — шаг внутрь)** — переходил по строкам кода и входил в вызовы методов для глубокого анализа.
- **Наблюдение за значениями переменных с помощью подсказок и окон инспекции** — проверял содержимое массивов, строк и переменных прямо во время остановки выполнения.
- **Проверка стека вызовов** — использовал окно стек вызовов, чтобы следить за текущим местом выполнения и возвратами из методов.
- **Использование навигации по коду** — перемещался по коду для уточнения логики выполнения и анализа поведения программы.
- **Инспектирование переменных через всплывающие подсказки** — смотрел значения переменных прямо при наведении мыши.
- **Выполнение до курсора** — частично использовал “Выполнить до текущей строки”, чтобы быстро перейти к нужной части кода.

Эти методы позволяют эффективно контролировать и анализировать выполнение программы для поиска и исправления ошибок.
